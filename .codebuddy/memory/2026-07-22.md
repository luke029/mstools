- **UNTRUSTED signature 问题根因（重要）**：
  - LuCI 上传安装走 `/usr/libexec/package-manager-call install /tmp/upload.apk`，最终执行 `apk add /tmp/upload.apk`，**不带 --allow-untrusted**
  - OpenWrt 25.12 的 apk-tools 3.0.5 对本地 .apk 文件也要求签名验证，无 SIG 块或无信任密钥则报 UNTRUSTED
  - 即使本地 openssl 验证签名正确，apk 仍拒收（RSA 签名格式兼容性问题）
  - 结论：不签名是最简单的方案，路由器需改 LuCI 脚本加 `--allow-untrusted`
